# 流体惯性鼠标 (Fluid Inertia Cursor) 动效复刻技术文档

## 1. 动效核心定义

本动效不是静态的指针替换，而是一个**基于鼠标移动速度驱动、具有物理惯性反馈的流体粒子系统**。全程**无点击交互**参与。

### 1.1 视觉与交互表现 (Visual & Interaction)

* **黑色流体拖痕（核心主体）**
    * **触发**：伴随鼠标移动实时生成。
    * **形态**：纯黑实心，边缘呈现不规则、粗糙的锯齿状，模拟墨水或高粘度流体。
    * **动态**：拖痕的长度与厚度与鼠标当前移动速度成正比（移动越快，拖拉越长）。
* **惯性与消散（灵魂细节）**
    * **惯性前冲**：当鼠标骤停时，拖痕前端不会瞬间冻结，而是带有动量地顺着原方向继续向前滑行、挤压。
    * **平滑收缩**：在滑行减速的同时，整个流体形态会向中心聚拢、变圆润，最终平滑缩小直至完全消失。
* **白色定位焦点（辅助防丢）**
    * **触发逻辑**：仅当鼠标处于移动状态，且被大面积生成的黑色流体包裹时显现。
    * **作用**：在纯黑区域中提供精准的高对比度定位点，防止用户视觉迷失。
    * **消失逻辑**：鼠标静止且流体消散时，白点同步平滑缩小至消失。

## 2. 底层物理逻辑模型 (Physics Engine Logic)

底层的数据模型需要建立一个微型的“粒子物理引擎”来驱动视觉表现：

1.  **速度测量**：实时计算鼠标移动的 X/Y 轴速度矢量。
2.  **粒子生成**：鼠标移动时高频生成“流体粒子”。
3.  **速度继承与脱离**：粒子生成瞬间继承鼠标当前速度，随后**立即脱离**鼠标坐标控制，成为独立个体。
4.  **阻尼衰减 (Friction)**：在每一帧渲染中，独立粒子的速度需乘以摩擦系数（例如 `v *= 0.9`），实现平滑减速的“惯性”效果。
5.  **生命周期 (Lifecycle)**：每一帧缩减粒子的尺寸或透明度，直至尺寸归零后销毁，实现流体的平滑“消散”闭合效果。

## 3. React + TypeScript 落地技术方案：Canvas 内部接管 (工业级最佳实践)

为了在现代前端工程中完美复刻这种需要极高渲染性能、复杂边缘计算（锯齿状）以及成百上千粒子同时高频更新的动效，**放弃纯 DOM 渲染，将核心物理与绘制逻辑交由 Canvas 2D API 接管** 是唯一的正解。

### 3.1 核心实现架构

此方案将 React 的组件化优势与 Canvas 的高性能渲染完美结合：

1.  **构建组件外壳**：创建一个名为 `<FluidCursor />` 的 React 组件，其内部仅返回一个铺满全屏、固定在顶层且 `pointer-events: none`（确保不遮挡底部元素交互）的透明 `<canvas>`。
2.  **面向对象封装 (TypeScript)**：在组件外部或单独的文件中，利用 TS 构建一个独立的 `Particle` 类。该类应包含：
    * **属性**：坐标 (`x`, `y`)、速度矢量 (`vx`, `vy`)、大小 (`radius`)、生命值 (`life`)。
    * **方法**：更新状态的 `update()` 方法（计算摩擦力与生命衰减）和负责绘制的 `draw(ctx)` 方法。
3.  **生命周期挂载**：在组件的 `useEffect` 中获取 Canvas 上下文 (Context)，并绑定对全局 `window` 的 `mousemove` 事件监听，用于实时计算速度并向数组中 `push` 新生成的粒子实例。
4.  **接管渲染循环**：在 `useEffect` 内部启动 `requestAnimationFrame` 函数。这个循环函数将接管一切：每一帧清空画布，遍历所有存活的粒子执行 `update()` 和 `draw()`，并清理掉死亡的粒子。

### 3.2 方案优势

* **视觉极致还原**：可以轻松支持海量的粒子渲染，通过自定义不规则的 Canvas 绘图路径，能够完美复刻视频中那种粗糙、有机的流体/墨水边缘。
* **性能拉满**：Canvas 专职负责高频的物理计算和像素级绘制，彻底避开了 React 虚拟 DOM 的对比和浏览器 DOM 树的重排/重绘 (Reflow/Repaint)，确保动效在 60fps 下丝滑运行。

---

## 4. 总结与建议

复刻这个动效的关键在于**转变前端思维**：不要把它看作是一个普通的 React UI 组件，而是在 React 环境中嵌入了一个微型的 2D 流体模拟器。采用 **外层 React 负责生命周期 + 内层 Canvas 负责物理计算与高频渲染** 的架构，不仅能原汁原味地重现视频中具有生命力和物理质感的动效，更能保障整个前端项目在加入酷炫动效后的运行性能。
